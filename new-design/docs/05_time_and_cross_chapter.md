# 时间维度 + 跨章节缝合 · 说明文档

在单章四维度(场景/人物/物品/地点)之上,增加第五维度【时间】,并新增【跨章节】层:把同一实体跨章归一,把同一人物分散在多章的事件缝成连续的个人时间线。

## 一、时间维度(章节内)

文件:`时间分析_提示词.txt`(单 pass,引用人物清单的 id)。

抽取两类信息:
- **time_expressions**:文中字面出现的时间词(clock钟点 / duration时长 / relative顺序词),逐字锚点可校验。
- **events**:关键事件,带【双时间轴】:

| 字段 | 含义 |
|---|---|
| narrative_order | 叙述顺序(文本里第几个被讲到) |
| story_order | 故事顺序(真实发生先后) |
| is_flashback | 叙述靠后但故事靠前 = 闪回 |
| participants | 涉及人物(引用人物 id,施事+受事都算) |
| abs_interval | 与上一事件的绝对间隔,**仅原文明确时填**,否则 null |
| storyline | 所属人物线/情节线 |

**关键纪律**:绝对时间只来自原文字面,没写明=null,绝不推算编造。多线叙事用 storyline 区分,不混成一条全局绝对轴。

**验证**:文本A(倒叙开篇)双轴正确分离——按故事序排列时叙述序为乱序 `[2,3,4,7,8,9,5,1,6,10]`,4 个闪回正确识别,participants 挂接 10/10,时间词锚点全命中。

## 二、跨章节缝合

文件:`跨章缝合_cross_chapter.py`(确定性代码 + 歧义报告)。输入各章的 {characters, events, items, locations} JSON,输出:

### 1. 全局实体归一
章内局部 id(每章从1开始)→ 全局实体 id。用 name/alias 交集做并查集合并:
- 主名相同 = 高置信合并
- 仅别名/部分名重叠 = 合并但**记入歧义报告**待人工确认

验证中:华剑雄(第1章#1=第2章#1)、司机=老赵(跨章别名重叠)正确归一并标"跨章同一人"。

### 2. 个人时间线缝合
每个全局人物一条时间线:把所有章里 participants 含该人物的事件,按(章节顺序, 章内 story_order)缝成连续序列。闪回保留 is_flashback 标记。

验证中:华剑雄的 14 个事件从第1章(长春回忆)缝到第2章(宪兵部),排出连续个人线。

### 3. 跨人物同步点
被 ≥2 个全局人物共享的事件 = sync_point,多条人物时间线在此对齐(如"华剑雄得知武田在刑讯室"对齐华剑雄+武田两条线)。

### 4. 歧义报告
`ambiguities` 列出所有"靠别名重叠合并、非主名相同"的归并,供人工确认。验证中正确报出"特命全权大使 vs 裴仁基"(原文未明说同一人)这个老大难歧义,不擅自合并。

## 三、设计决策(对应需求)

- **多人事件归属**:进所有参与者的线,共享同一 event_id —— 这样人物线之间能通过共享事件对齐。
- **缝合程度**:相对连续顺序 + 原文明确的绝对间隔(abs_interval)+ 跨人物同步点。无原文依据不推算绝对间隔。
- **叙事连续 vs 故事连续**:用 narrative_order / story_order 双轴分别表达,is_flashback 标记两者错位。

## 四、已知局限

- **绝对时间**:这类小说极少给日历日期,abs_interval 多为 null(只有"一个月""才一天"这类相对时长能填)。这是文本特性,非缺陷——方案诚实保留 null 而非编造。
- **跨章实体归一靠名称匹配**:同名不同人(两个"小李")或同人完全不同名且无别名桥接的情况,自动归一会漏或错 → 全部进歧义报告,靠人工终裁。
- **story_order 跨章是相对的**:章间绝对先后靠章节物理顺序假定;若书本身打乱章节顺序(非线性叙事),需人工指定章节的故事先后。

**总原则**:时间词/事件经锚点校验可信;双轴与闪回标注准确;绝对间隔只认原文;跨章归一信主名匹配、复核别名匹配与所有歧义报告项。
