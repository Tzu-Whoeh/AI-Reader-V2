# 脏人名合并 · 实体归一模块说明

应对盗版/OCR/爬取网文里同一人名的"脏变体":符号噪音、错别字、繁简、全半角、零宽字符。是跨章实体归一的前置增强模块。

文件:`实体归一_entity_normalize.py`

## 三层逻辑

### 第一层:符号归一(确定性,零风险)
`normalize_name(s)` 把名字归一:
- NFKC 兼容分解 + 全角转半角(华剑雄→华剑雄)
- 剥离零宽字符(\u200b \u200c \u200d \ufeff \u2060)
- 剥离所有标点/符号/空白(下划线、间隔号·、星号、引号、括号等)
- 繁转简(内置简版映射;生产建议接 opencc 覆盖全字表)

效果:`华_剑_雄`、`华 剑 雄`、`华·剑·雄`、`華劍雄`、`华​剑​雄`(含零宽)、`*华剑雄*` 全部 → `华剑雄`。归一后同名直接自动合并。

### 第二层:错别字合并(编辑距离 + 上下文佐证)
对归一后仍不同但相似的名字,用**编辑距离**(适配中文短名,比字符比率敏感)+ **上下文佐证**分级:
- 归一后编辑距离 == 1(差一字,典型错别字):
  - 上下文吻合(role 词重叠 / 共同场景 / 共同地点 / 共同关系方)→ **auto 自动合**
  - 上下文不支持 → **review 待人工确认**(防"华剑熊茶楼老板"误并入主角)
- 编辑距离 == 2 或中等相似 → review
- 其余 → no

**核心思想**:名字差一字时,让上下文当决定性证据——这就是"配合身份场景"判断,而非纯字形匹配。

### 第三层:重名防误合(role 严重互斥检测)
应对"同名不同人":同名默认合并(宽松),但当两人 role 命中**不同身份大类且无交集**时降级 review。
身份大类(可扩展):特务谍报 / 军警官员 / 商贩平民 / 学界文人 / 服务侍从。
- 同名 + 同大类(特务/心腹)→ auto
- 同名 + 严重互斥(特务/茶楼掌柜)→ review「疑似重名不同人」
- 一方 role 无法归类 → 不算冲突,auto(宽松)

## 分级总表

| 名字关系 | 上下文 | 判定 |
|---|---|---|
| 归一后同名 | 不冲突 | auto |
| 归一后同名 | role 大类严重互斥 | review |
| 编辑距离1(错字) | 吻合 | auto |
| 编辑距离1(错字) | 不支持 | review |
| 编辑距离2/中等相似 | — | review |
| 不相似 | — | no |

## 接入跨章缝合

在 `跨章缝合_cross_chapter.py` 的实体归一环节,把原来的"名字交集"判断替换为:先 `normalize_name` 再 `classify_merge`。auto 的并入同一全局实体,review 的进 `ambiguities` 报告,no 的分开。

## 已知局限

- **繁简映射是简版**:内置仅覆盖常见字,生产环境应接 opencc(`pip install opencc`)做全字表转换。
- **身份大类词表需按作品扩充**:不同题材的职业词不同,`_ROLE_CLASSES` 应按实际文本补充。
- **编辑距离对调字序不敏感**:"剑雄"vs"雄剑"距离2,会进 review 而非自动合(通常也确实该人工看)。
- **错字+无上下文**:孤立出现、没有 role/场景信息的脏名字,只能进 review,无法自动判定。

**总原则**:符号归一确定可信;错字与重名一律"上下文吻合才自动合,否则进报告",绝不靠名字字面单独决定。
