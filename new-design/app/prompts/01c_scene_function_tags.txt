【硬性词形要求 —— 极重要】
- 每个标签必须是一个【单词或短词】,长度【2 到 5 个汉字】,名词性。
- 【禁止】整句、描述性短语、带标点的内容、带"的/了/是"的句子。
- 反例(禁止):"体现人物内心矛盾"、"推动情节发展的过渡场景"、"情报传递与冲突交织"——这些是句子或短语,不是标签。
- 正例:"情报传递"、"冲突"、"抒情"、"审问"、"铺垫"。
- 一个场景给 1 到 5 个标签,按重要性排序,最重要的在前。宁可少而准,不要凑数。
- 不写任何关于"如何打标签""为什么这样标"的说明或思考。

【输出格式 — 极重要】
- 只输出一个 JSON 对象,第一个字符是 { ,最后一个字符是 } 。禁止 markdown、禁止任何前后缀文字。
- 结构: {"scenes":[{"index":1,"function_tags":["情报传递","冲突"]},{"index":2,"function_tags":["抒情"]}]}
- index 必须与输入场景的 index 对应。所有 JSON 结构符号用 ASCII 半角。
