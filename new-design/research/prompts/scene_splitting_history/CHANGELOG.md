# 场景拆分 prompt 演进 CHANGELOG

本目录存 v2→v8 的版本存档。定稿版(v8 内容)已落到 `new-design/prompts/01_scene_splitting.txt`。
此处保留迭代史,记录每版**改了什么、为什么、效果**,供以后避免重走弯路。
(v1 = 仓库原始版,即被 v8 替换前 main 上的版本,核心是"通常 4-6 个"数量锚点。)

## v1 → v2:删数量锚点
- **改**:删掉"通常 4-6 个"。
- **因**:数量锚点对长复杂文本封顶(C样本该6段切5)、对短文本下界推着过度拆。
- **效果**:C 解封(能拆6),但**矫枉过正**——常规章 ch44 从5段碎成12段。

## v2 → v3:"叙事目的"判据
- **改**:判据改为"叙事目的转换才切",不是功能(现实/回忆/内心/动作)每次切换都切。
- **效果**:方向对(A/B不再过拆),但长对话仍偏碎;35b 比 27b 碎得多。

## v3 → v4:"一场连续交谈=一个目的"(场次优先)
- **改**:首要标准改为"场次"——同时间同地点同组人的连续交谈算1段,话题换了不切。
- **效果**:doc3→4、doc4→3(对);但 **ch44 合过头**(质问→调情被焊死成1段)。

## v4 → v5:目的性质突变例外
- **改**:场次优先 + "互动根本性质反转(对抗↔亲密)仍切"的例外。
- **效果**:ch44 修好(5段),但 doc3/doc4 又退;典型"治一个惊一个"。

## v5 → v6:四类客观边界 + cut_reason
- **改**:判据重构为四类客观边界(loc/cast/time/turn),每段标 cut_reason 使切点可审计。
- **效果**:方向对(边界客观可验证),但 turn/cast/time 软边界仍漂移(稳定性测试 doc4/C 漂)。

## v6 → v7:只认两类硬边界
- **改**:砍掉 turn(性质反转)和 cast(人物进出),只留 loc(地点)+ time(往事回放)。
  内心独白明确不独立成段。软边界判断下放事件层。
- **效果**:C 从一路漂(4/3/5)变铁稳(4/4/4);ch44 段数稳(2/2/2)。
  仅剩 ch44 一个切点的 loc/time 标签摆动。

## v7 → v8:loc/time 优先级消歧(定稿)
- **改**:加一句——一处同时像 loc 又像 time(思绪转别时空别处)统一标 time;
  loc 只给身体真实移动。
- **效果**:ch44 标签摆动消除(start-time 三跑一致)。**6文本5个完全稳,定稿。**

---

## 贯穿始终的教训
1. **不要用数量描述粒度**("4-6个"是万恶之源)。用可逐字指认的客观边界。
2. **软边界(需解读的)别硬塞进场景层**,下放到事件层。
3. **不要加后处理**消漂移——反而放大(详见 findings.md §6)。
4. **prompt 迭代会"治一个惊一个"**——出现这种来回时,说明判据维度不对,
   要换更客观的维度(如 v5→v6 从"叙事目的"换到"客观边界"),而非继续微调。
